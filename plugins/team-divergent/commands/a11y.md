---
description: Dauntless — accessibility audit + fix (semantic HTML, keyboard paths, focus management, ARIA, contrast, reduced-motion) to WCAG AA.
argument-hint: [file, component, or page — defaults to recent UI changes]
---

Launch the `dauntless` subagent (Team Divergent — Senior Frontend Engineer) to audit and fix accessibility on: **$ARGUMENTS** (default: recently changed UI files).

- **Semantic HTML first** (a button is a `<button>`); full keyboard operability with a logical focus order; focus managed on route/dialog open/close; a visible focus indicator on every interactive element.
- Correct **ARIA only where semantics can't do the job** (wrong ARIA is worse than none); accessible names for every control, icon-button, and image; honor `prefers-reduced-motion`.
- **Contrast** to WCAG AA; never color as the only signal.
- **Verify:** do a keyboard-only walkthrough of the flow, run lint + build, and add tests where the framework supports it.

Boundaries: frontend files only. Relay issues found and how each was fixed.
